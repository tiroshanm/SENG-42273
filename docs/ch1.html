üîç The Web of Documents: A Machine's Dilemma

Understanding the limitations of keyword-based search

üî¥ Key Issue: Keyword Search vs. Meaning

Traditional search engines (analyzed in sw2) rely on simple keyword matching and link structure (PageRank), which lacks semantic understanding.

üìÑ Limitation 1: Human-Optimized Content

The existing World Wide Web (Web 2.0) is optimized for human consumption. Content is presented visually, relying on human intelligence for interpretation.

Documents only give machines instructions about PRESENTATION, not UNDERSTANDING.

üîó Limitation 2: No Semantic Context

The Internet is constructed such that it is oblivious to actual information content. Web browsers, servers, and search engines cannot distinguish between different types of content.

Example: Search for "Python" provides no context regarding whether it is the snake üêç, the programming language üíª, or the Monty Python group üé≠

‚ö†Ô∏è The Challenge

This requires human effort to disambiguate terms. Machines see the text, but cannot reliably infer the relationships or the intended meaning of the content.

üéØ The Root Problem

All three main Web activities (Search, Integration, Web Mining) suffer from the same underlying issue:

Computers can PRESENT information but cannot UNDERSTAND information

üí≠ Real-World Impact

Consider booking an airline ticket:

  • Step 1: Check flight schedules on airline website
  • Step 2: Compare prices on travel agent site
  • Step 3: Process payment through payment service

Problem: You must manually integrate these three separate Web services!

Tim Berners-Lee's Definition

"The Semantic Web is an extension of the current Web in which information is given well-defined meaning, better enabling computers and people to work in cooperation."

üéØ Core Goal

Add machine-understandable semantics (structured meaning) to data on the Web, transforming it from documents to data.

üìñ The Extension

An extension of the current Web where information is given well-defined meaning, enabling computer-human cooperation.

üîß Mechanism: Metadata

Metadata provides the essential link between page content and content meaning (as stated in sw1), transforming unstructured data into a structured asset.

üåç Transformation

From a collection of isolated documents to a coherent, queryable global database of structured knowledge.

üîë What Makes It Different?

  • Semantic Web: Documents enriched with structured metadata about their meaning
  • Result: Machines can make intelligent decisions and process data at global scale

üí° Key Benefit

By adding semantics through metadata, we enable:

  • Automated information integration
  • Intelligent search with understanding
  • Web mining at scale
  • Machine reasoning and inference

üîó Tim Berners-Lee's Linked Data Principles

The foundation for publishing and connecting semantic data

Guidelines for Semantic Data Publishing

These principles define how to publish and connect data on the web using semantic standards, forming the foundation of the Semantic Web.

1

üè∑Ô∏è Naming Things

Use URIs to denote everything‚Äîentities, concepts, and relationships. This ensures a unique, global identifier for every piece of knowledge.

2

üåê Accessibility

Use HTTP URIs so entities can be looked up (dereferenced) by machines and humans alike, reinforcing the web nature of data.

3

üìã Standard Format

When a URI is looked up, provide useful, structured data using W3C Standards (e.g., RDF, RDFS, OWL).

4

üîó Connections

Include links (relationships) to other URIs to enable discovery and navigation of related data across different sources.

üéØ Why These Principles Matter

Together, these four principles create a web of data that is:

  • Globally accessible: Any machine can look up and understand resources
  • Interoperable: Standards ensure different systems can work together
  • Connected: Data from different sources can be integrated seamlessly
  • Extensible: New data can be added without breaking existing connections

üí° Knowledge Graphs: Structured Reality and Reasoning

Building blocks for machine-understandable knowledge

üìä What is a Knowledge Graph?

A Knowledge Graph (KG) is a network that models real-world entities and their relationships in a formal structure, often built upon RDF Triples.

üß± The Building Block: RDF Triple

The core is the Triple (Subject-Predicate-Object), which formalizes a single fact.

Structure: Every statement follows the same pattern, making it machine-readable and processable.

Example Triple:

Subject: CityPark:CentralGreen
Predicate: isLocatedInZone
Object: Zone:ResidentialA
‚úÖ Benefit 1: Explicit Context
Provides explicit, queryable context and relationships, moving beyond simple data tables or unstructured documents.
‚úÖ Benefit 2: Machine Reasoning
Enables sophisticated machine reasoning and complex querying (via SPARQL) to infer new, non-explicit facts‚Äîkey to our "Resilient City" goal.

üîç From RDF to Metadata to Knowledge Graphs

The progression:

  • RDF (Resource Description Framework): The language for expressing triples
  • Metadata: Structured data about data, expressed in RDF
  • Knowledge Graphs: Collections of interconnected RDF triples forming a queryable knowledge base

üéØ Why Knowledge Graphs for Resilient Cities?

In our project, we'll model city infrastructure using a KG to:

  • Track relationships between parks, sensors, and zones
  • Automate risk assessment based on sensor data
  • Generate maintenance alerts through logical inference
  • Enable complex SPARQL queries for city management

üîç Interactive Demo: The Search Problem

Understanding why keyword matching fails

The Classic Example: Searching for "SOAP"

SOAP is a W3C standard for Web services (Simple Object Access Protocol). But when you search for it using traditional keyword matching...

Try It Yourself!

Search Results: ~128,000,000 results

1. Buy Ivory Soap - Best Detergent Online

Get the best deals on dish soap and laundry detergent...

2. Soap Opera Daily - Latest Episodes

Watch your favorite soap operas online...

3. Handmade Natural Soaps - Organic Products

Shop our collection of artisanal soaps made with essential oils...

4. SOAP Protocol Specification - W3C

Simple Object Access Protocol (SOAP) Version 1.2 - Technical Specification...

5. Soap Making Tutorial - DIY Home Guide

Learn how to make your own natural soap at home with our step-by-step guide...

‚ùå The Problem

Search engines implement keyword matching. As long as a document contains the keyword, it's included in results‚Äîregardless of context or meaning.

Result: You get detergents, soap operas, DIY tutorials, and only after sifting through multiple pages do you find the W3C's SOAP specifications!

‚úÖ How Semantic Web Solves This

With semantic markup, each document would declare its type:

  • Document 1: Type = "Product/Cleaning"
  • Document 2: Type = "Entertainment/TVShow"
  • Document 4: Type = "Technical/Specification/WebService"

Your search would specify: Search for "SOAP" WHERE Type = "Technical/Specification"

Result: Only relevant technical documents returned!

üí≠ Discussion Point

Question: Why can't search engines distinguish between different meanings of the same word?

Hint: Think about what information is available to the search engine. Does the HTML contain semantic information about the document's subject matter?
