üîç Keyword Search vs. Meaning and Context

The fundamental limitation of traditional search

üî¥ Key Issue: Keyword Search vs. Meaning

Traditional search engines (analyzed in Lecture 1) rely on simple keyword matching and link structure (PageRank), which completely lacks semantic understanding.

The Search Process

  1. User enters keywords
  2. Engine matches keywords in index
  3. Returns ALL pages with those keywords
  4. Ranks by frequency and links

‚ùå What's Missing?

  • Context: Is this about learning? A job posting? A research paper?
  • Intent: Does the user want tutorials? Documentation? News?
  • Relationships: Connection between words in the query
  • Semantics: Meaning of the words in this specific combination

üìä From Lecture 1: The SLR Camera Problem

Remember searching for "SLR" cameras:

  • Got vendor sites (high keyword frequency)
  • Missed "single lens reflex" pages (different words)
  • Missed pages about "shutter speed" (related property)
  • No way to filter by intent (buying vs. learning)

ü§ñ Machines See Text, Not Meaning

The fundamental conclusion about machine understanding

The Fundamental Conclusion

Machines see the text, but cannot reliably infer the relationships or the intended meaning of the content.

üí° The Semantic Web Solution Revisited

Connecting problems to Lecture 1's solution

üîó From Lecture 1: The Three-Part Solution

Now that we deeply understand the problems, let's revisit HOW the Semantic Web solves them:

‚úÖ Solution 1: Common Vocabularies (Ontologies)

Create structured, machine-readable definitions of concepts and their relationships.

Photography Domain Example:
‚Ä¢ Defines: SLR ‚â° SingleLensReflex
‚Ä¢ Relates: ShutterSpeed is property of SLR
‚Ä¢ Hierarchies: SLR is type of Camera

‚úÖ Solution 2: Semantic Markup (Metadata)

Page owners explicitly link their content to ontology concepts.

<link rel="help" href="photography-ontology.owl">

This tells machines: "My page about 'single lens reflex' means SLR in the photography ontology"

‚úÖ Solution 3: Smart Agents/Crawlers

Crawlers that can read ontologies and make inferences.

Inference Example:
1. Page discusses "shutter speed"
2. Markup links to photography ontology
3. Ontology: ShutterSpeed is property of SLR
4. Crawler infers: Page is about SLR cameras!
5. Indexes under "SLR" even though word never appears
